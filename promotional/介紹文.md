## OCM Sup 係乜？

OCM Sup（OpenClaw Memory Supreme）係一套 AI 長期記憶系統，建基於 OpenClaw 生態。

佢唔只係儲存資料——佢會：
- **優先排序** — 重要嘢記住，垃圾慢慢忘記
- **關聯知識** —唔係散落嘅筆記，係有關係嘅知識網
- **自動維護** — 唔需要你手動執嘢，佢自己搞得掂
- **自我修復** — 發現矛盾會自動標記，唔會繼續錯落去

你用落，就係感覺到你嘅 AI 助手「記得嘢」。

## 核心功能

### 🧠 智能記憶分层

```
對話即時記憶
        ↓
短期記憶（7日）
        ↓
長期記憶（按內容重要性衰減）
        ↓
核心知識（永久保留）
```

### ⚡ 三路搜索

結合三種搜尋方式，確保每次都搵到最相關嘅資料：

| 搜尋方式 | 擅長 |
|---------|------|
| **BM25** | 關鍵字精確匹配 |
| **向量搜索** | 語義相似內容 |
| **知識圖譜** | 相關實體擴展 |

### 🔒 信心評分

每條資訊都有信心分數（0-1）：
- 多次確認 → 高分
- 90日未確認 → 慢慢衰減
- 新舊矛盾 → 自動標記舊版本為過時

### 🔄 主動矛盾檢測

寫入新資訊前，系統會問：
> 「呢個同現有知識有冇衝突？」

如有，舊資訊自動標記為 `stale`，新資訊正常寫入。

### ⏱️ 自動遗忘

唔同內容有唔同嘅 decay rate：
- 每日對話 → 14日後衰減
- 項目進度 → 45日
- 核心決策 → 90日
- 重要原則 → 永久保留

## 開始使用

### 步驟一：Clone OCM Sup

```bash
git clone https://github.com/st007097-coder/ocm-sup.git
cd ocm-sup
```

### 步驟二：運行健康檢查

```bash
python3 scripts/ocm-sup-health-check.py
```

確認所有組件正常運作。

### 步驟三：設定自動化

```bash
# 設定每日自動記憶整合
openclaw cron add \
  --name "OCM-Sup Consolidation Loop" \
  --cron "30 4 * * *" \
  --tz "Asia/Hong_Kong"
```

### 步驟四：注入決策記憶

你每一個決定，阿星都會記得並標記為「重要」。
